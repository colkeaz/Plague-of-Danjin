# Three-class system with skill trees and save persistence (v2)

The eight issues from v1 have been addressed. Multi-target detection now recognizes `STUN_MULTI_TARGET` and `POISON_MULTI_TARGET`. TIME_WARP skips the next enemy turn via a `timeWarpActive` flag. Arcane Affinity and Arcane Mastery reductions are applied in `processMultiTargetAction`. Immortal Stand uses a dedicated `immortalStandActive` flag independent of `autoReviveActive`. Fortress and Warlord's Fury use status-effect-based temporary buffs that revert on expiry. `spellCostReductionBonus` and `plagueBearerActive` are serialized in SaveData and restored.

**Watch for:** Passive skills (Arcane Mastery, Immortal Stand, Plague Bearer) have 0 cost and 0 cooldown, allowing unlimited re-casting — Arcane Mastery stacks +5 per cast, enabling trivial cost reduction to zero (confirmed). `immortalStandActive` is not serialized, so the passive deactivates on save/load (confirmed). The silent-removal behavior of `StatusManager.addEffect` for non-stackable effects can permanently leak stat buffs if overlapping durations occur in future balance changes (possible).

**Verdict**: NEEDS_CHANGES

## High-level view

Two new issues emerge from the fixes. Passive skills (`PASSIVE_COST_REDUCTION`, `PLAGUE_BEARER`, `IMMORTAL_STAND`) are implemented as normal skills with 0 cost and 0 cooldown, meaning they occupy a skill slot and can be re-cast every turn. Arcane Mastery uses `+= 5` per cast, so it stacks infinitely. Immortal Stand can be re-cast after its one-time trigger to reset `immortalStandUsed`, giving the Knight unlimited safety nets.

The save/load fix for `spellCostReductionBonus` and `plagueBearerActive` was applied, but `immortalStandActive` was missed — the same pattern, the same bug.

`StatusManager.addEffect` silently removes existing same-type effects without firing STATUS_EXPIRED. The current cooldown/duration pairing prevents overlap for DEF_BUFF and ATK_BUFF, but this invariant is undocumented and fragile against future changes.

<summary>Issues (3)</summary>

1. **Arcane Mastery infinite stacking** (confirmed) — `PASSIVE_COST_REDUCTION` adds +5 to `spellCostReductionBonus` on every cast with 0 cost/0 cooldown. A Mage can reduce all spell costs to 0 in a few turns. Guard the increment with an `if (spellCostReductionBonus == 0)` check or set the cooldown to `Integer.MAX_VALUE` after first use.
2. **Immortal Stand reusable / not persisted** (confirmed) — `immortalStandActive` is not in SaveData (lost on reload), and re-casting the skill resets `immortalStandUsed = false`, giving unlimited 50%-HP heals at the 20% threshold. Add `immortalStandActive` to SaveData serialization and prevent re-activation after trigger (`immortalStandUsed` should block re-cast).
3. **Buff stat leak on effect replacement** (possible) — `StatusManager.addEffect` silently removes existing same-type effects without firing STATUS_EXPIRED. If future balance changes allow ATK_BUFF or DEF_BUFF to overlap (e.g., by reducing cooldown below duration), the first buff's stat change is never reverted. Consider firing an expiry event on forced replacement, or documenting the "cooldown >= duration" invariant as a requirement.

## Passive skills as infinitely-castable abilities

Arcane Mastery, Plague Bearer, and Immortal Stand are defined as skills with `manaCost = 0` and `cooldownTurns = 0`. They sit in the player's skill tree alongside normal combat skills. Nothing prevents the player from selecting them every turn.

```java
case PASSIVE_COST_REDUCTION:
    this.spellCostReductionBonus += 5;
    break;
```

Each cast unconditionally adds 5 to the reduction. A Mage who picks Arcane Mastery at wave 10 can cast it 3 times over the next few turns to reach -15 total reduction, making Fireball (15 MP base) completely free. Immortal Stand resets `immortalStandUsed = false`, which re-arms the 50%-HP safety net after it was already consumed — the Knight can re-cast every few turns to maintain unlimited protection.

The simplest fix: add a guard in `applySkillEffect` that checks whether the effect has already been applied and short-circuits. Alternatively, set `cooldownTurns` to `Integer.MAX_VALUE` so the skill never comes off cooldown after first use.

## `immortalStandActive` missing from save/load

The fix commit added `spellCostReductionBonus` and `plagueBearerActive` to SaveData — both permanent skill flags. `immortalStandActive` follows the same lifecycle (set once by casting the skill, needed for `checkImmortalStand()` to function) but was not included. A Knight who activates Immortal Stand, saves, and reloads will find the passive silently disabled.

## Buff reversal and the replacement pattern

`StatusManager.addEffect` silently discards same-type effects via `removeIf` without triggering expiry events. If a future change reduces cooldowns or adds other sources of DEF_BUFF, the replaced effect's stat contribution would persist permanently. The current design relies on an undocumented invariant: cooldown must be >= duration for status-backed stat buffs. Worth adding a comment or assertion enforcing this.
