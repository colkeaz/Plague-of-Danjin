# QTE Boss Events, Procedural Audio, and Save/Meta-Progression

Three interlocking systems added in a single phase: a Quick-Time Event layer that interrupts boss combat at HP thresholds, a fully procedural 8-bit audio engine generating all sounds and music at runtime from waveform math, and a save/continue system with cross-run unlockable bonuses. The QTE and save systems bolt onto the existing CombatEngine state machine; the audio layer sits in the view package and subscribes to game events. All audio and game state serialization avoid external libraries, using manual PCM/WAV construction and hand-rolled JSON parsing respectively.

**Watch for:** Save restoration cannot accurately restore player HP below max (confirmed); the hand-rolled JSON parser fails on nested brackets in string values (likely); `MetaProgression` turn counting only increments on `PLAYER_BASIC_ATTACK`, missing spell-cast turns (confirmed); temp WAV files written to disk on every `SoundGenerator.createSound()` call accumulate if deletion fails (possible).

## High-level view

The QTE system is cleanly layered: `QTEPattern` holds static data for seven boss thresholds, `QTEManager` handles real-time input matching and timing, `CombatEngine.checkQTEThreshold()` gates on HP values and fires a state transition, and `QTEScreen` renders the overlay. Thresholds are hardcoded against fixed HP values that don't account for the Shatter modifier reducing boss max HP by 20%, causing incorrect trigger points when Shatter is active.

The audio engine generates all 18 SFX and 5 music loops eagerly at construction time, writing temporary WAV files to local storage for each Sound object. `MusicManager` crossfades between tracks using `Sound.loop()` with volume ramping, a pragmatic workaround for libGDX's `Music` requiring actual file assets. `Sound` has a platform-dependent maximum duration, so longer tracks may be silently truncated on some backends.

The save system writes to `~/.plague_of_danjin/` using `Gdx.files.absolute()` and a bespoke JSON serializer/parser. The restoration path (`CombatEngineRestorer`) rebuilds engine state by starting a fresh game and adjusting stats, which means it cannot increase max HP or precisely set current HP below max — every continued game starts at full HP. The `MetaProgression` listener's turn counter only increments on basic attacks, making the "fastest victory" metric and `swift_boots` unlock trivially easy for spell-heavy builds.

<summary>Issues (8)</summary>

1. **QTE thresholds ignore Shatter modifier** — Boss HP thresholds are hardcoded (e.g., Lich at 225/150/75) but the Shatter modifier reduces boss max HP by 20%. If Shatter is active, a Lich has 240 max HP and the 75% threshold (225) fires almost immediately. Compute thresholds as percentages of actual max HP instead of absolute values. (confirmed)

2. **Save restoration cannot reduce HP below max** — `CombatEngineRestorer` calls `fullRestore()` then has no way to set HP to `saveData.getHp()` because `GameCharacter` exposes no `setHp()` and `takeDamage()` applies defense. Add a `setHp(int)` method or use a bypass-defense damage path. (confirmed)

3. **MetaProgression turn counter misses spell turns** — `currentRunTurns` only increments on `PLAYER_BASIC_ATTACK` events. Turns where the player casts a spell are not counted, making `fastestVictoryTurns` always lower than actual turns taken. Increment on all action processing or use `CombatEngine.turnCounter` directly. (confirmed)

4. **Temp WAV file accumulation on failure** — `SoundGenerator.createSound()` writes a temp file, loads it, then deletes. If `Gdx.audio.newSound()` throws, the file is never deleted (the catch returns null). Use a try-finally to ensure cleanup. (confirmed)

5. **Hand-rolled JSON parser fragile on brackets in strings** — `parseStringList()` finds the first `]` after `[` to determine array boundaries. If any string value earlier in the JSON contains a `]` character, the parser truncates. Use balanced-bracket counting or escape-aware scanning. (likely)

6. **Sound.loop() duration limit** — libGDX's OpenAL backend (LWJGL3) buffers `Sound` objects fully in memory and some platforms cap at ~10 seconds. The music tracks are 3-5 seconds currently but extending them later will hit silent truncation. Document this constraint. (possible)

7. **MetaProgression not persisted on abnormal exit** — `saveMetaProgression()` is only called at run end. If the game crashes mid-run, all stats accumulated via the event listener (kills, damage) are lost. Consider periodic meta-save at wave transitions alongside the run save. (likely)

8. **Listener accumulation on repeated GameScreen creation** — Each `new GameScreen(game, engine)` registers `sfxManager`, `metaProgression`, and `messageLog` as listeners on the engine. When QTEScreen creates a new GameScreen on return, duplicate listeners accumulate, causing doubled SFX playback and corrupted meta-progression stats. Remove listeners in `hide()` or `dispose()`. (confirmed)

## QTE threshold calculation vs. Shatter modifier

The `getQTEPatternForThreshold()` method compares `enemy.getHp()` against hardcoded absolute values: GoblinChieftain at 60, GoblinKing at 75/37, BoneColossus at 100, Lich at 225/150/75. These correspond to percentage thresholds of the original max HP. However, `applyShatterEffect()` calls `enemy.reduceMaxHpByPercent(20)` on wave setup when the Danjin's Heart Shatter modifier is active. A Lich under Shatter has 240 max HP, making the 75% threshold (225) fire almost immediately when HP drops below starting value. The 25% threshold at 75 HP now represents 31% of actual max.

```java
// Current: hardcoded
if (currentHp <= 225 && !triggeredQTEThresholds.contains("Lich_75")) {
    return QTEPattern.lich75();
}

// Should be: percentage-based
int threshold75 = (int)(enemy.getMaxHp() * 0.75f);
if (currentHp <= threshold75 && ...) { ... }
```

## Save restoration fidelity gap

`CombatEngineRestorer.restoreFromSave()` starts a fresh engine, then attempts to adjust the player to match saved state:

```java
// After fullRestore(), player.getHp() == saveData.getMaxHp()
// No way to reduce to saveData.getHp() without:
// - A setHp() method (doesn't exist)
// - takeDamage() which subtracts defense first
// - reduceMaxHp() which permanently lowers the cap
```

Every continued run starts at full HP regardless of saved state, which is a gameplay advantage on Continue vs. dying and restarting.

Additionally, the `iron_constitution` unlock equips a runtime-synthesized armor item with MAX_HP+15. If that item name is saved, `CombatEngineRestorer.findItemByName()` searches `ItemRegistry.getAllItems()` which won't contain it — the item only exists when created in `applyUnlocks()`. Restored games lose the +15 max HP bonus.

## MetaProgression turn-counting asymmetry

```java
case PLAYER_BASIC_ATTACK:
    currentRunTurns++;
    break;
```

`CombatEngine.processPlayerAction()` increments its own `turnCounter` unconditionally on every action. A player who casts spells for 40 of 60 turns will report `currentRunTurns = 20` to MetaProgression while the engine says 60. The `swift_boots` unlock ("victory in under 60 turns") becomes trivially easy with spell-heavy builds.

## Procedural audio temp-file lifecycle

```java
FileHandle file = Gdx.files.local(fileName);
file.writeBytes(wavData, false);
Sound sound = Gdx.audio.newSound(file);  // if this throws...
file.delete();                             // ...this never runs
return sound;
```

The catch block returns null without deleting the file. The `tempFileCounter` is static and resets on JVM restart, so new runs will typically overwrite old temps, but concurrent game instances would not.

## Listener registration lifecycle in GameScreen

Each time a `GameScreen` is constructed, it registers listeners on the engine:

```java
engine.addListener(messageLog);
engine.addListener(this);
engine.addListener(sfxManager);
engine.addListener(meta);
```

The QTE flow creates a `QTEScreen`, and when the QTE completes, `QTEScreen` creates a *new* `GameScreen(game, engine)` passing the same engine. The old GameScreen's listeners are never removed. After one QTE, the engine has duplicate `sfxManager` and `metaProgression` listeners. After 3 Lich QTEs, there are 4 sets. The `sfxManager` playing sounds multiple times is audible; the `metaProgression` multi-counting kills corrupts stats.

## JSON parser fragility with bracket characters

Both `SaveData.parseStringList()` and `MetaProgression.parseStringList()` find array boundaries with:

```java
int bracketStart = json.indexOf('[', colonIndex);
int bracketEnd = json.indexOf(']', bracketStart);
```

This finds the *first* `]` after the opening `[`, not the matching one. If a field before the target array contains a `]` in its string value (e.g., a player name like `"Hero]X"`), parsing silently returns wrong data. `escapeJsonString()` escapes quotes and backslashes but not brackets, so they pass through literally into the JSON output.
