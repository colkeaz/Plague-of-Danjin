# Non-linear dungeon hub with branching story areas (v3)

This change transforms a linear 20-wave gauntlet into a hub-and-spoke world map with four branching dungeon areas, narrative events with mechanical consequences, and three ending variants driven by player choices. The v2 review found two issues: the Plague Gardens absorb choice promised poison without applying it, and the VictoryScreen double-counted the Lich's Offer in the corruption formula. Both are fixed. The absorb path now applies a permanent POISON(999, potency 5) status effect matching the existing poison pattern used throughout the codebase. The ending formula removes the redundant `eventChoices` increment for Lich acceptance, relying solely on the `danjinHeartAbsorbed` flag (+2 corruption) to score that decision.

**Watch for:** No blocking concerns remain. The two v2 fixes are correctly implemented and the build passes clean.

**Verdict**: APPROVED

## High-level view

The poison fix uses `StatusEffect(StatusType.POISON, 999, 5, "Corrupted Tree")` — the same constructor signature as every other status application in the codebase. Duration 999 makes it effectively permanent (matching the Plague Survivor's REGEN 999 pattern), and potency 5 delivers the promised 5 damage/turn. The player now pays a real ongoing cost for the +20 ATK, making the purify/absorb choice a genuine tradeoff.

The ending formula fix removes the `lichChoice == 0` corruption increment, leaving `absorbed` as the sole corruption source for the Lich's Offer at +2 points. The threshold for Corrupted Victor is still `>= 2`, so accepting the Lich's Offer alone locks in the corrupted ending (2 points) while tree-absorb alone (1 point) does not. This produces three distinct reachable endings depending on the combination of choices made across both events.

## Ending reachability after the fix

With the corrected formula, the corruption and purity scores break down cleanly:

- Lich accept: +2 corruption (via `absorbed` flag)
- Tree absorb: +1 corruption (via `eventChoices`)
- Tree purify: +1 purity
- Lich refuse: +1 purity
- Goblin free: +1 purity
- Paladin blessing: +1 purity
- Scepter shatter: +1 purity

Corrupted Victor triggers at `corruptChoices >= 2` — achievable only by accepting the Lich's Offer (with or without tree absorb). Pure Hero triggers at `pureChoices >= 3` — requires refusing the Lich, purifying the tree, and at least one of goblin/paladin/scepter. Everything else falls to Pragmatic Survivor. All three endings are reachable through distinct play patterns, and no single choice dominates the formula.

## Poison persistence across save/load

The 999-turn poison applied by the Corrupted Tree fix will serialize correctly via the existing `StatusManager` save path in `CombatEngineRestorer`, which already handles status effects as `"type:duration:potency:source"` strings. The `parseStatusEffect` method reconstructs them on load. Since `POISON` is an existing `StatusType` and the serialization format is type-agnostic, no additional save/load changes were needed for this fix.
