# Non-linear dungeon hub with branching story areas

This change transforms a linear 20-wave gauntlet into a hub-and-spoke world map with four branching areas (Goblin Warrens, Bone Cathedral, Plague Gardens, Lich's Throne), each containing 4-6 encounters, one narrative event, and a boss. The model layer introduces Area, AreaData, Encounter (Builder pattern), WorldState, and AreaEvent. The controller layer adds WorldManager (area data initialization, encounter routing, key/progress tracking) and AreaEncounterGenerator (enemy instantiation with area-based scaling). CombatEngine gains a `storyMode` flag and dual-mode handling so classic 20-wave play remains intact. Four new screens (WorldMapScreen, AreaScreen, RestShrineScreen, AreaEventScreen) handle navigation, and VictoryScreen gains three ending variants driven by player choices. Save/load is extended with 9 new fields serialized as string-encoded lists.

**Watch for:** Story mode has no path to VictoryScreen after defeating the Lich (confirmed). The Lich's Offer event grants +50/+50 stats and marks `danjinHeartAbsorbed` without applying `permanentDamagePerTurn`, creating an asymmetry with classic mode's Danjin Heart event (confirmed). Toxic atmosphere deals 3 damage to the player's own enemies too тАФ likely intentional flavor, but undocumented and potentially confusing to players (confirmed).

## High-level view

WorldState gating is simple тАФ three areas are always open, Lich's Throne unlocks at 3 keys тАФ which gives players genuine non-linearity without risk of soft-locking.

CombatEngine's dual-mode split is the riskiest structural decision. A single `storyMode` boolean gates all branching in `handleStoryModeEncounterComplete`, `processAreaMechanic`, and `startEncounter`. The engine is accumulating responsibilities that will resist further extension.

The victory path is broken in story mode. After the final Lich boss is defeated, `handleStoryModeEncounterComplete` transitions to `WORLD_MAP`, not `VICTORY`. GameScreen routes `WORLD_MAP` to `WorldMapScreen`. No screen in the chain checks whether Lich's Throne is complete and triggers `VictoryScreen`. Players will return to the hub after killing the final boss with no epilogue.

The event choice system records decisions and uses them in VictoryScreen's ending calculation, but the Lich's Offer event doesn't apply `permanentDamagePerTurn` when absorbing power. In classic mode, absorbing Danjin's Heart costs +30/+30 stats at the price of 10 damage/turn permanently. In story mode's Lich's Offer, the player gets +50/+50 with no ongoing cost тАФ only the `danjinHeartAbsorbed` flag is set, which triggers the Corrupted Victor ending path but not the per-turn penalty the flag was designed to carry.

Save/load serializes event choices as `"eventName:choiceIndex"` strings parsed with `lastIndexOf(':')`. All current event names are colon-free, so this works today. A future event name containing a colon would split on the wrong boundary.

## Missing victory transition in story mode

When `handleStoryModeEncounterComplete` runs after the Lich is killed, `advanceAfterEncounter` marks Lich's Throne as complete and `worldManager.isAreaComplete(current)` returns true. The method then sets `currentState = GameState.WORLD_MAP`. GameScreen's state handler for `WORLD_MAP` routes unconditionally to `new WorldMapScreen(game, engine)`. WorldMapScreen has no logic to detect that the run is won. The result: the player defeats the final boss and sees the hub again with Lich's Throne marked `[DONE]`.

The fix: in `handleStoryModeEncounterComplete`, after calling `worldManager.advanceAfterEncounter()`, check if the completed area is `Area.LICHS_THRONE` and set `currentState = GameState.VICTORY` instead of `WORLD_MAP`.

## Lich's Offer event vs classic Danjin's Heart asymmetry

In `AreaEventScreen.makeChoice` for `LICHS_THRONE`, accepting the offer runs:

```java
player.upgradePower(50);
player.upgradeDefense(50);
engine.getRunModifiers().setDanjinHeartAbsorbed(true);
```

The classic-mode equivalent in `EventRoomManager.applyDanjinHeartChoice` also calls `modifiers.setPermanentDamagePerTurn(10)`, which CombatEngine applies every turn via:

```java
if (runModifiers.isDanjinHeartAbsorbed() && runModifiers.getPermanentDamagePerTurn() > 0) {
    runModifiers.applyPermanentDamagePerTurn(hero, ...);
}
```

Because `permanentDamagePerTurn` defaults to 0, the story-mode version gets all the benefit and none of the cost, yet both set the same flag that VictoryScreen uses to determine the "Corrupted Victor" ending. If the design intent is identical risk/reward, add the per-turn cost. If story mode intentionally makes the choice more generous, the ending logic is correct but the CombatEngine's per-turn check is silently inactive тАФ either way, one side needs alignment.

## Area mechanic scope тАФ toxic atmosphere hitting enemies

In `processAreaMechanic`, the `TOXIC_ATMOSPHERE` case deals 3 damage to the player and then iterates all living enemies dealing 3 damage to each. VOID_INSTABILITY hits only the player (2 damage). CONSECRATED_GROUND has no per-turn effect. The asymmetry means the player's strategy in Plague Gardens can lean on passive enemy attrition тАФ 3 HP/turn compounds across every enemy across every turn. If all 3 enemies in encounter 4 take 3 damage/turn for 5 turns, that's 45 free damage before accounting for poison stacking from PlagueElementals.
