# Non-linear dungeon hub with branching story areas (v2)

This change transforms the linear 20-wave gauntlet into a hub-and-spoke world map with four branching areas, each with encounters, a narrative event, and a boss. The v1 review found four issues. Two were fixed: VictoryScreen is now reachable after the Lich defeat, and the Lich's Offer event now applies `permanentDamagePerTurn(10)`. The toxic atmosphere behavior (damaging enemies too) was confirmed intentional. The colon-delimited serialization was accepted as minor tech debt.

**Watch for:** The Plague Gardens "Absorb" choice promises "poison 5/turn" in the UI text but only grants +20 ATK without applying any poison status effect (confirmed). The VictoryScreen epilogue double-counts the Lich's Offer choice — it's recorded in `eventChoices` AND triggers the `danjinHeartAbsorbed` flag, adding 3 corruption points instead of the intended 2 for a single decision (confirmed).

**Verdict**: NEEDS_CHANGES

## High-level view

Two new concerns emerge from closer inspection of the event system. The Plague Gardens absorb choice has a UI/implementation mismatch: the choice text says "+20 ATK, take poison 5/turn" but the code only calls `upgradePower(20)` with no poison application. And the VictoryScreen ending calculation adds corruption for the Lich's Offer via `eventChoices` (`lichChoice == 0` adds 1) *and* via the `absorbed` flag (adds 2), giving a single choice triple weight in the ending formula.

<summary>Issues (2)</summary>

1. **Plague Gardens absorb choice missing poison** — The choice text reads "+20 ATK, take poison 5/turn" but `makeChoice` for `PLAGUE_GARDENS` index 1 only calls `player.upgradePower(20)`. No poison `StatusEffect` is added and no `permanentDamagePerTurn` is set. Either apply a permanent poison (e.g., `player.getStatusManager().addEffect(new StatusEffect(StatusType.POISON, 999, 5, "Corrupted Tree"))`) or change the choice description to remove the poison promise.
2. **Lich's Offer double-counted in ending calculation** — VictoryScreen's `buildStoryEpilogue` counts the Lich's Offer event choice (`lichChoice == 0` increments `corruptChoices`) *and* then checks `absorbed` (`if (absorbed) corruptChoices += 2`). Both conditions are set by the same player action in `AreaEventScreen.makeChoice`. A player who accepts the offer gets 3 corruption points from one decision. Either remove the `eventChoices` check for the Lich (since `absorbed` already captures it with higher weight) or reduce the `absorbed` bonus to +1.

## Plague Gardens absorb choice — promise without implementation

In `AreaEventScreen.makeChoice`, the `PLAGUE_GARDENS` case for index 1 runs:

```java
resultText = "You absorb the tree's power. Dark energy floods your veins. You feel stronger, but tainted.";
if (player != null) {
    player.upgradePower(20);
}
```

The choice text presented to the player is: "Absorb its power (+20 ATK, take poison 5/turn)". No `StatusEffect` of type `POISON` is applied, no `permanentDamagePerTurn` is set, and no other mechanism delivers the 5 damage/turn. The player gets a free +20 ATK with no downside, making this choice strictly dominant over "Purify the tree" (which only removes the 3 dmg/turn toxic atmosphere that also hurts enemies). Classic mode's Danjin Heart uses `permanentDamagePerTurn` for ongoing cost, and the story mode Lich's Offer now correctly applies it — the Plague Gardens absorb path should use the same pattern or have its choice text corrected.

## VictoryScreen ending formula double-counts corruption

In `buildStoryEpilogue`:

```java
Integer lichChoice = choices.get("The Lich's Offer");
if (lichChoice != null && lichChoice == 0) corruptChoices++;  // +1 from event

if (absorbed) corruptChoices += 2;  // +2 from flag
```

Both `lichChoice == 0` and `absorbed == true` are consequences of the same user action in `AreaEventScreen.makeChoice` for `LICHS_THRONE` index 0. The net effect is +3 `corruptChoices` from accepting one offer. Since the "Corrupted Victor" threshold is `corruptChoices >= 2`, accepting only the Lich's Offer (without the Plague Gardens absorb) is enough to lock in the corrupted ending at 3 points. This makes the other corruption opportunity (Plague Gardens absorb) irrelevant to ending determination — the formula can never produce a non-corrupted ending once the Lich's Offer is accepted, regardless of all other choices being pure.
