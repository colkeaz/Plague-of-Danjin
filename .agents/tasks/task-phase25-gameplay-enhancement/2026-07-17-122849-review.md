# Gameplay enhancement layer for Plague of Danjin (v2)

All nine issues from the v1 review have been addressed. The fix commit introduces six new `SkillEffect` enum values with corresponding logic in `Player.applySkillEffect`, a `reflectHandler` listener in CombatEngine for BoneColossus damage, poison immunity enforcement in PlagueGoblin, ENRAGE stat tracking with reversal, and `isPlayerDeadAfterReviveCheck()` for the Resurrection auto-revive. The project compiles cleanly.

**Watch for:** Drain Life heals based on raw damage before defense reduction, making it consistently stronger than intended against high-DEF targets (confirmed). Berserker Rage reversal can push defense below zero or attack below original values if other modifiers were applied during the buff window (confirmed). BoneColossus reflect fires on its own `takeDamage(int)` path but player attacks route through `takeDamage(int, Element)` — the shield-break check works only because the elemental overload delegates to the non-elemental path (confirmed, not a bug, but fragile coupling).

**Verdict**: APPROVED

## High-level view

Each unique skill mechanic now gets its own `SkillEffect` enum value and dedicated `case` block. Drain Life heals 50% of the computed (pre-mitigation) damage. Assassinate conditionally picks 6x or 2x based on `target.getHp() < target.getMaxHp() * 0.3`. Blood Pact checks `getHp() <= 20` to prevent self-kill. Berserker Rage adds an ENRAGE status, with `CombatEngine` applying +100%/−50% on detection and reverting on expiry — but the reversal arithmetic operates on absolute values snapshotted at buff time, which creates edge cases if stats change during the window.

The BoneColossus reflect uses a `reflectHandler` listener registered on each enemy. The shield duration was corrected from 99 to 1 (breaks on the next hit, triggering reflect). The reflect path depends on `takeDamage(int, Element)` delegating to `takeDamage(int)` where BoneColossus's override lives — a fragile contract that should be documented.

PlagueGoblin checks `hasPoisonImmunity()` before applying poison, scoped to its own `attack()` method. Future poison sources would need independent immunity checks.

## Drain Life heal calculation versus actual damage dealt

```java
case DRAIN_LIFE:
    int drainDamage = (int)(getTotalAttackPower() * skill.getDamageMultiplier());
    target.takeDamage(drainDamage, skill.getElement());
    int healAmount = drainDamage / 2;
    if (healAmount > 0) {
        this.heal(healAmount);
    }
    break;
```

`drainDamage` is the raw value passed to `takeDamage`. Inside `GameCharacter.takeDamage(int)`, defense reduces this: `finalDamage = (int)(damage - damage * (defense / 100.0))`. Against BoneColossus (25 DEF), a 60-power player with 2.0x multiplier computes `drainDamage = 120`, deals 90 actual damage, but heals 60. The spec says "heal 50% of damage dealt" — ambiguous whether "dealt" means before or after mitigation. If intended as post-mitigation, `takeDamage` would need to return the actual HP lost, requiring a refactor to `GameCharacter`.

## Berserker Rage stat reversal edge cases

The engine snapshots `enrageAtkBonus = hero.getTotalAttackPower()` at application time (base ATK + equipment). On reversal, it calls `upgradePower(-enrageAtkBonus)`, which modifies the base `attackPower` field with no lower bound. If the player's equipment contributed to `getTotalAttackPower()` at snapshot time, the reversal over-subtracts from the base stat alone.

For defense: `enrageDefReduction = hero.getDefense() / 2`. If a curse reduces DEF during the buff window, the reversal `upgradeDefense(enrageDefReduction)` adds back the full original amount — defense ends higher than before the rage. In practice, defense reductions during rage are rare (only Danjin's Curse at wave boundaries), but the arithmetic is unsound.

## Reflect handler ordering dependency

```java
public void takeDamage(int damage, Element attackElement) {
    double multiplier = Element.getMultiplier(attackElement, this.element);
    int modifiedDamage = (int)(damage * multiplier);
    takeDamage(modifiedDamage);  // <-- delegates to the non-elemental version
}
```

BoneColossus overrides `takeDamage(int)` to detect shield-break. This only works because the elemental version delegates downward without intercepting the shield check itself. If future code adds shield handling in `takeDamage(int, Element)` before the delegation, BoneColossus's override never fires and reflect breaks silently.
